# Polaris UI/UX Improvement Proposal

Based on a thorough review of the current Polaris frontend components (like `TodayView`, `Dashboard`, and `ActivityCard`), the current UI is functional but lacks the "premium, polished" aesthetic that modern productivity tools demand. 

We can completely overhaul the aesthetic to make it a more engaging, serene, and habit-forming experience.

## 🎨 1. StitchMCP: The "Celestial" Design System

I have utilized **StitchMCP** to generate a brand new design system and high-fidelity mockups for Polaris. The new design language, dubbed **"Polaris Celestial"**, moves away from strict boxy constraints and embraces a dark-mode editorial aesthetic.

### Key Visual Upgrades:
*   **Color Palette (The Indigo Pulse):** Moving away from generic grays. We are using deep space tones (`#0e0e13` for backgrounds) with vibrant, glowing Indigo (`#6366f1`) for active states and progress.
*   **The "No-Line" Rule:** Removing harsh 1px borders in favor of subtle background color shifts and `backdrop-blur` (glassmorphism) to define depth.
*   **Typography:** Using **Inter** with tight tracking for large headers to create an editorial, high-end feel, rather than a standard web app look.
*   **Cards & Elevation:** Instead of flat cards, we treat elements as physical layers of tinted glass.

## 🏗️ 2. Component Refactoring Plan

To implement this design, we will need to refactor several core components in Phase 2:

1.  **`ActivityCard.tsx`**: Update from a standard bordered box to a glassmorphism card (`bg-gray-900/40 backdrop-blur-md`). Improve the inline value prompt to appear seamlessly rather than pushing content down.
2.  **`Badge.tsx`**: Replace solid color backgrounds with soft, semi-transparent tints (e.g., `bg-indigo-500/10 text-indigo-400`).
3.  **`TodayView.tsx`**: Transition from a vertical list to the horizontal **Kanban-style** layout shown in the mockup, separating "Planned", "Completed", and "Skipped" into distinct sweeping columns.
4.  **`Dashboard.tsx`**: Implement the 30-day continuous Heatmap and the horizontal "Stat Cards" (Completed, Rate, Streak) using the new glow effects.

## 🗄️ 3. Developer Data UI: Prisma Studio

While StitchMCP handles the user-facing UI, we also need to improve how *we* interact with the data during development.

Currently, checking if an activity parsed correctly or tracking database records involves writing raw SQL or logging to the console.

**Recommendation:** We should utilize **Prisma Studio** (via the `prisma-mcp-server`).
Prisma Studio provides a beautiful, modern admin UI locally. It will allow us to:
*   Visually inspect the exact relationship between an AI-parsed `Activity` and its parent `Goal`.
*   Quickly edit test data without writing seed scripts during the UI prototyping phase.
*   Filter and paginate through thousands of logged activities smoothly.
